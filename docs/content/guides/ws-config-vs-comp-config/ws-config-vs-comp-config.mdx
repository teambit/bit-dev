Bit is composed of Aspects. It has no "core". For this reason, configurations can only be set on Aspects.

Aspects configured in the [Workspace configuration](/workspace/workspace-configuration) are applied on the workspace whereas, Aspects configured in the [Component configuration](/components/component-config) are applied on the component.
The two configs may produce similar effects on components in that Workspace, but only in the context of that specific Workspace.

Components [exported](/workspace/exporting-components) from one Workspace and [imported](/workspace/importing-components) into another, will lose their configuration, since the new Workspace does not have the same Aspects applied on it.
This is not the case for Aspects that are applied on Components (i.e, via the component config).

The examples below show the (fictional) `my-org.my-scope/snapshots` aspect being configured and applied using both ways. This Aspect creates UI component snapshots. It also defines a certain set of component files (`**/snapshot.*`) as [development files](/components/dev-files).
One of this workspace's components is `ui/button`. As you can see, in both cases `button.snapshot.png` is successfully defined as a development file. This will be persisted when the component is [tagged](/components/tags) or [snapped](/components/snaps).

However, once the `ui/button` component is imported into a different workspace, one that does not have the `my-org.my-scope/snapshots` aspect in its Workspace configuration, it will no longer be affected by that Aspect.
This will result in redefining the `button.snapshot.png` file as a standard file, instead of a dev file. This change will be persisted the next time this component is versioned.

#### `my-org.my-scope/snapshots` applied via the workspace config:

```json
//
{
  "my-org.my-scope/snapshots": {
    "useDevFiles": true
  }
}
```

```bash
$ bit show ui/button

┌───────────────────┬───────────────────────────────────────────────────────────────────────────┐
│ id                │ my-org.my-scope/ui/button                                                 │
├───────────────────┼───────────────────────────────────────────────────────────────────────────┤
│ ...               │ ...                                                                       │
├───────────────────┼───────────────────────────────────────────────────────────────────────────┤
│ files             │ index.ts                                                                  │
│                   │ button.tsx                                                                │
│                   │ button.scss                                                               │
│                   │ button.snapshot.png                                                       │
├───────────────────┼───────────────────────────────────────────────────────────────────────────┤
│ dev files         │ button.snapshot.png (my-org.my-scope/snapshots)                           │
├───────────────────┼───────────────────────────────────────────────────────────────────────────┤
│ extensions        │ teambit.react/react                                                       │
│                   │ teambit.component/dev-files                                               │
│                   │ teambit.compositions/compositions                                         │
│                   │ teambit.pkg/pkg                                                           │
│                   │ teambit.docs/docs                                                         │
│                   │ teambit.envs/envs                                                         │

│ ...               │ ...                                                                       │
└───────────────────┴───────────────────────────────────────────────────────────────────────────┘
```

Notice how the above output does not list the `my-org.my-scope/snapshots` aspect as one of the `extensions` used by that component.

#### `my-org.my-scope/snapshots` applied via the component config:

```json
{
  "teambit.workspace/variants": {
    "{ui/**}": {
      "my-org.my-scope/snapshots": {
        "useDevFiles": true
      }
    }
  }
}
```
