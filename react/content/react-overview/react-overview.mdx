# React Overview

React is teambit's Component Development Environment (Env) for React.
It enables you to quickly set up a dev environment that follows best practices for React component development.

React can also be used as a good starting point for your own customized Env as it provides a comprehensive API that makes it easy to extend.

## Using the React env

To use React Env, apply it on the relevant [variants](/docs/workspace/variants) (component selections) in your [Workspace config](/docs/workspace/workspace-configuration):

```json title="workspace.jsonc"
{
  "teambit.workspace/variants": {
    "{ui/**}": {
      "teambit.react/react": {}
    }
  }
}
```

## Creating Components

React env provides a set of component templates that can be used to speed-up and standardize your component development:

To list all available templates:

```bash
bit templates
```

- `react` a basic React UI component.
- `react-context` a react context component.
- `react-hook` a react hook component.
- `mdx-component` an MDX component.
- `react-component-js` a basic React component in JS.
- `react-env` boilerplate for a customized environment configuration.

Use any of these templates with the `bit create` command:

```bash
bit create react ui/my-welcome
```

### Default React versions

By default, the React env sets the following dependency versions for components using it:

- **react** (peer dependency): `^16.8.0 || ^17.0.0`
- **react-dom** (peer dependency): `^16.8.0 || ^17.0.0`
- **@types/react**: `^17.0.8`
- **@types/react-dom**: `^17.0.5`
